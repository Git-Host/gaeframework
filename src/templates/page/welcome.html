Welcome!

Your new project based on GAE framework is launched.

Below we discuss about how to configure your project and launch your site in the web.

Create new project

Create new empty project with two directories: src (for source code) and doc (for project documentation). We recommend you use version control system (Subversion or Mercurial).

Download latest version of GAE framework. Unpack it. Copy contents to your project src directory.

Configure your project

Change project settings in apps/config.yaml file.

Set appspot ID

Open file app.yaml and set your application ID like this application: my-project-id. If you have not an application ID than register new application on appspot.com

Change urls mapping

By default, site homepage is static page, located in temlates/page/welcome.html. You can change this behavior by changing your urls section in project configuration file.

Each url rule consist of:

  • url - defines what path should be used for match requested page
  • run - define request handler in format app_name.handler_name, where handler_name is function from file apps/app_name/controllers.py
  • map - define additional urls mapping for use to the continue search requested path. We need define application name, and this is equal to use urls section from apps/app_name/config.yaml file
  • arg - additional arguments, passed to request handler

You can use only run or map option separately. Don't use this both options in the same url rule!

Customize standard applications

Many useful applications you can found in apps directory.

You can change design and behavior of all applications without changes in his source code.

You can redefine:

  • templates for change application design. Copy some template files from directory apps/app_name/templates to templates/app_name and change this files
  • configuration for change application behavior. Copy appropriate options from apps/app_name/conig.yaml to urls: app_name section in the file apps/config.yaml

Create your applications

Learn what is already created in apps directory and make similar application with your custom logic in this directory.
